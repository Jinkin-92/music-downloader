## 📋 执行摘要

### 背景说明
音乐下载器应用已实现完整的批量下载功能，包括多源搜索、智能匹配、结果切换等核心功能。但由于 `pyqt_ui/main.py` 存在大量缩进错误，应用无法启动。

用户在了解现有功能后，提出两项功能增强需求，旨在提升用户体验和操作效率。

### 当前优先级
1. **P0 (Critical)**: 修复启动问题 - 解决缩进错误
2. **P1 (High)**: 表格内快捷切换功能
3. **P2 (Medium)**: 匹配模式可调功能

---

## 🔍 现有功能分析

### 已实现功能模块

#### 1. 数据模型 (`pyqt_ui/batch/models.py`)

**核心类**:
- `MatchCandidate`: 单个搜索候选结果
  - 存储歌曲信息（歌名、歌手、专辑、时长等）
  - 相似度分数 (`similarity_score`)
  - 原始SongInfo对象引用

- `BatchSongMatch`: 单首歌的所有匹配结果
  - `current_match`: 当前选中的匹配
  - `current_source`: 当前匹配的源（如 "NeteaseMusicClient"）
  - `all_matches`: 字典 `{source: [candidates]}` - 存储所有源的搜索结果
  - `match_source_type`: 匹配来源（AUTO/USER_SELECTED）

**关键方法**:
```python
get_all_candidates_from_current_source() -> List[MatchCandidate]
    # 获取当前源的所有候选结果

get_all_candidates() -> List[MatchCandidate]
    # 获取所有源的所有候选结果（按相似度降序排序）

switch_to_candidate(candidate: MatchCandidate, source_type: MatchSource)
    # 切换到指定的候选结果
```

#### 2. UI组件 (`pyqt_ui/batch/match_switcher_dialog.py`)

**MatchSwitcherDialog 对话框**:
- **尺寸**: 900x600px
- **模式**: 模态对话框
- **两个Tab页**:

  **Tab 1: "按源分组"**
  - 按音乐源分组显示候选结果
  - 显示每个源的候选数量
  - 列: ☐ | 歌名 | 歌手 | 专辑 | 时长 | 相似度
  - 相似度颜色编码:
    - ≥80%: 深绿色
    - 60-80%: 深黄色
    - <60%: 红色

  **Tab 2: "所有候选（按相似度排序）"**
  - 跨源显示所有候选结果
  - 按相似度降序排序
  - 列: ☐ | 歌名 | 歌手 | 专辑 | 来源 | 时长 | 相似度

- **交互功能**:
  - 点击候选行 → 启用"切换到此匹配"按钮
  - 预览按钮 → 显示候选详细信息
  - 切换按钮 → 发送 `match_changed` 信号

#### 3. 主UI集成 (`pyqt_ui/main.py`)

**批量结果表格**:
- **列定义**: ☐ | # | 歌名 | 歌手 | 来源 | 操作 | 相似度
- **第5列（操作列）**:
  - 已匹配歌曲: 显示"切换"按钮
  - 未匹配歌曲: 显示"重搜"按钮

**关键方法**:
```python
on_switch_match(original_line: str)  # line 880
    # 打开切换匹配对话框

on_match_changed(original_line: str, new_candidate)  # line 897
    # 处理匹配切换，更新UI显示
    # 更新表格中的来源、相似度等列
```

#### 4. 匹配算法 (`pyqt_ui/batch/matcher.py`)

**SongMatcher 类**:
- **相似度计算**: 使用 `difflib.SequenceMatcher`
- **匹配阈值**: `SIMILARITY_THRESHOLD = 0.6` (60%)
- **组合分数**: 歌名相似度 × 0.7 + 歌手相似度 × 0.3
- **标准化**: 转小写、移除特殊字符

## 💡 新功能需求

### 需求1: 表格内快捷切换 (P1 - High)

#### 用户目标
在批量结果表格内直接切换匹配结果，**无需打开对话框**，提升操作效率。

#### 用户故事
> 作为批量下载用户，我希望能够在结果表格内快速切换不同的匹配结果，而不需要每次都打开对话框，这样可以更高效地管理大量歌曲的匹配。

#### 功能需求

**FR1.1: 表格内切换控件**
- 在"操作"列（第5列）添加下拉选择器
- 或在"来源"列（第4列）直接切换
- 显示当前源的候选数量提示（如 "QQ音乐 (3)"）

**FR1.2: 候选导航**
- 方案A: 下拉选择器
  - 点击展开当前源的所有候选
  - 显示候选列表（歌名 - 歌手 - 相似度）
  - 选择后立即切换并更新UI

- 方案B: 左右箭头按钮
  - 添加 [<] [>] 按钮
  - 点击遍历当前源的所有候选
  - 显示当前候选索引（如 "2/5"）

**FR1.3: 跨源切换**
- 如果当前源没有满意结果，提供快速切换到其他源的选项
- 如：下拉菜单底部显示 "切换到其他源..." → 打开完整对话框

**FR1.4: 保留完整对话框**
- "切换"按钮仍保留，用于复杂切换场景
- 或右键菜单添加"高级切换..."选项

#### 非功能需求

**NFR1.1: 性能**
- 切换操作应在 100ms 内完成
- 不阻塞UI线程

**NFR1.2: 易用性**
- 键盘导航支持（方向键、Enter）
- 视觉反馈清晰（高亮当前选择）
- 撤销功能（Ctrl+Z 恢复上一次切换）

**NFR1.3: 兼容性**
- 与现有对话框功能共存
- 不影响现有数据结构

### 需求2: 匹配模式可调 (P2 - Medium)

#### 用户目标
允许用户自定义匹配严格程度，适应不同的使用场景（精确匹配 vs 宽松匹配）。

#### 用户故事
> 作为批量下载用户，我希望能够调整匹配的严格程度，因为有时候我需要完全精确的匹配（如正式发布版本），而有时候我愿意接受相似的歌曲（如现场版、 remix等）。

#### 功能需求

**FR2.1: 匹配模式预设**
提供3种预设模式：
- **严格模式**: 相似度 ≥ 90%（歌名和歌手都要求高度匹配）
- **标准模式**: 相似度 ≥ 60%（当前默认）
- **宽松模式**: 相似度 ≥ 40%（允许较大差异）

**FR2.2: 自定义阈值**
- 允许用户手动调整相似度阈值（0-100%）
- 分别调整歌名和歌手的权重（默认: 70% + 30%）

**FR2.3: UI位置**
- 在批量搜索区域添加"匹配设置"面板
- 或在批量搜索对话框中添加设置选项

**FR2.4: 模式切换**
- 实时生效（无需重新搜索）
- 显示当前模式和阈值
- 提供模式说明和推荐场景

#### 非功能需求

**NFR2.1: 数据持久化**
- 保存用户选择的模式到配置文件
- 下次启动自动加载

**NFR2.2: 性能**
- 切换模式不应触发重新搜索
- 实时过滤现有搜索结果

**NFR2.3: 用户体验**
- 提供清晰的模式说明
- 显示每个模式下的预计匹配数量

#### 技术实现建议

**配置存储** (`pyqt_ui/config.py`):
```python
# 新增配置项
MATCH_MODES = {
    'strict': {
        'name': '严格模式',
        'threshold': 0.9,
        'name_weight': 0.8,
        'singer_weight': 0.2,
        'description': '要求歌名和歌手都高度匹配'
    },
    'standard': {
        'name': '标准模式',
        'threshold': 0.6,
        'name_weight': 0.7,
        'singer_weight': 0.3,
        'description': '平衡准确性和召回率'
    },
    'loose': {
        'name': '宽松模式',
        'threshold': 0.4,
        'name_weight': 0.6,
        'singer_weight': 0.4,
        'description': '允许较大的差异，召回率更高'
    }
}

DEFAULT_MATCH_MODE = 'standard'
```

**UI控件**:
```python
# 在批量搜索区域添加
match_mode_group = QGroupBox("匹配设置")
mode_combo = QComboBox()
mode_combo.addItems([mode['name'] for mode in MATCH_MODES.values()])
mode_combo.setCurrentText(MATCH_MODES[DEFAULT_MATCH_MODE]['name'])

threshold_slider = QSlider(Qt.Orientation.Horizontal)
threshold_slider.setRange(0, 100)
threshold_slider.setValue(int(DEFAULT_MATCH_MODE * 100))

description_label = QLabel(MATCH_MODES[DEFAULT_MATCH_MODE]['description'])
```

**实时过滤**:
```python
def on_match_mode_changed(self, mode_name):
    """当匹配模式改变时，实时过滤结果"""
    mode_config = MATCH_MODES[mode_name]

    for line, song_match in self.current_batch_search_result.matches.items():
        # 重新计算符合当前阈值的候选
        filtered_candidates = [
            c for c in song_match.get_all_candidates()
            if c.similarity_score >= mode_config['threshold']
        ]

        # 更新UI显示
        if filtered_candidates:
            # 显示最佳匹配
            best_match = filtered_candidates[0]
            song_match.switch_to_candidate(best_match)
        else:
            # 标记为未匹配
            song_match.has_match = False

    self.refresh_batch_results_table()
```
