# 批量下载切换匹配功能 - 使用说明

## 功能简介

本功能允许你在批量搜索后，对不满意的结果进行切换，选择更好的匹配版本。

---

## 核心功能

### 1. 批量搜索配置

点击"批量搜索"后，会弹出配置对话框：

**搜索模式**：
- **快速模式**：找到第一个匹配就停止（速度快，~200秒/200首歌）
- **完整模式**：搜索所有源的所有匹配（速度较慢，~600秒/200首歌，但可选择更多版本）

**候选数量**：
- 每个源最多保存多少个候选（1-20个）
- 默认值：5个
- 完整模式建议：10个

---

### 4. 重搜功能

对于"未找到"的歌曲（显示为"-"），点击"重搜"按钮：

1. 弹出确认对话框
2. 系统会搜索所有源的10个候选（而非默认的5个）
3. 表格自动更新显示新结果

**使用场景**：
- 自动匹配不理想时重搜
- 想要更多候选时重搜
- 某个源搜索失败时重搜

## 使用示例

### 示例1：快速批量下载

1. 输入歌曲列表：
```
告白气球 - 周杰伦
没关系 - 容祖儿
七里香 - 林俊杰
```

2. 点击"批量搜索"
3. 在配置对话框中选择：
   - 搜索模式：快速模式
   - 候选数量：5个

4. 勾选歌曲并点击"下载"

---

### 示例2：完整搜索 + 切换匹配

1. 输入歌曲列表：
```
告白气球 - 周杰伦
没关系 - 容祖儿
七里香 - 林俊杰
```

2. 点击"批量搜索"
3. 在配置对话框中选择：
   - 搜索模式：完整模式
   - 候选数量：10个

4. 查看"告白气球"的搜索结果：
   - QQ音乐：85%相似度
   - 网易：78%相似度
   - 显示为"[切换]5个"

9. 勾选并下载

### 示例3：重搜未找到的歌曲

1. 批量搜索后，某首歌显示"未找到"（-）

2. 点击"重搜"按钮

3. 确认重搜

4. 系统搜索所有源的10个候选

## 注意事项

### 性能考虑

1. **200首歌批量下载**：
   - 快速模式：~200秒
   - 完整模式：~600秒
   - 建议：先使用快速模式，对不满意的再切换

2. **网络连接**：
   - 需要能访问音乐平台API（QQ、网易云等）
   - 网络不稳定可能导致搜索失败

3. **版权限制**：
   - 部分歌曲可能无法下载
   - 某些源可能需要VIP账号

### 常见问题

**Q1**: 程序启动失败？
- A: 检查Python版本是否为3.7+
- B: 检查PyQt6是否正确安装
- C: 查看错误日志 `logs/app.log`

**Q2**: 搜索结果为空？
- A: 检查是否选择了至少一个音乐源
- B: 确认歌曲列表格式（歌名 - 歌手）
- C: 检查网络连接

**Q3**: 切换后下载的还是旧版本？
- A: 确认在切换对话框中点击了"切换到此匹配"按钮
- B: 查看表格中的来源列是否更新
- C: 重新搜索该歌曲

**Q4**: 如何提高匹配准确率？
- A: 使用完整模式搜索所有源
- B: 增加候选数量（10-20个）
- C: 使用更准确的歌名和歌手名
