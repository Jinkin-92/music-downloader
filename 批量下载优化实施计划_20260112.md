## 背景与目标

### 当前功能状态

✅ **已实现**:
- 批量搜索（快速/完整模式）
- 通过对话框切换匹配（按源分组或跨源排序）
- 相似度颜色编码显示
- 重搜功能
- 用户已确认: "现在可以正常下载了"

### 优化目标

🎯 **用户体验提升**:
- 减少操作步骤（从3-4次点击减少到1-2次）
- 提供更灵活的匹配控制
- 加快常见操作流程

🎯 **功能增强**:
- **P1（高优先级）**: 表格内快速切换 - 在结果表格中直接切换匹配
- **P2（中优先级）**: 可调节匹配置信度 - 三种模式 + 自定义阈值

## 功能需求概述

### P1: 表格内快速切换（高优先级）

**用户痛点**:
- 当前需要点击"Switch"按钮打开对话框
- 对于简单的同源切换，对话框过于重量级
- 频繁切换时操作繁琐

**解决方案**:
- 在表格中添加快速切换控件
- 点击后直接显示候选菜单（无需对话框）
- 保留完整对话框用于复杂操作

**两种实现方案对比**:

| 特性 | 方案A: 下拉菜单 | 方案B: 箭头按钮 |
|------|----------------|----------------|
| 交互直观性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 空间占用 | 固定，较拥挤 | 按需，轻量 |
| 实现复杂度 | 高 | 中 |
| 与现有UI兼容性 | 可能冲突 | 较好 |

**推荐方案**: **混合方案**
- 在相似度列添加小箭头按钮"▼"
- 显示候选数量提示（如"▼5个"）
- 点击显示紧凑的下拉菜单
- 保留"Switch"按钮打开完整对话框

**目标性能**:
- 菜单打开时间: < 100ms
- 表格刷新时间: < 200ms

---

### P2: 可调节匹配置信度（中优先级）

**用户痛点**:
- 当前固定60%阈值，无法调整
- 想要更严格的匹配时需要手动筛选
- 想要宽松匹配时可能会漏掉结果

**解决方案**:
- 提供三种预设模式: 严格(≥90%)、标准(≥60%)、宽松(≥40%)
- 支持自定义阈值和权重
- 实时过滤，无需重新搜索
- 持久化用户偏好

**UI设计**:
```
┌─────────────────────────────────────┐
│ 匹配设置              [▼ 高级选项]  │
├─────────────────────────────────────┤
│ ○ 严格 (≥90%)                       │
│ ● 标准 (≥60%)  ← 默认               │
│ ○ 宽松 (≥40%)                       │
│                                     │
│ [高级选项 - 展开]                    │
│   自定义阈值: [───●──] 75%          │
│   歌名权重:   [───●──] 70%          │
│   歌手权重:   [───●──] 30%          │
└─────────────────────────────────────┘
```

**数据流设计**:
```
用户选择模式 → 应用阈值 → 过滤候选 → 刷新表格
      ↓
QSettings保存 → 下次启动自动加载
```

## 分阶段实施方案

### 阶段划分原则

✅ **独立可交付**: 每个阶段都有可验证的产出
✅ **用户价值驱动**: 优先实现高价值功能
✅ **技术风险可控**: 从简单到复杂，逐步验证
✅ **向后兼容**: 不破坏现有功能

### 阶段顺序

**为什么先做P2（可调节匹配置信度）？**

1. **实现相对简单**: 主要是UI和过滤逻辑
2. **不涉及复杂控件**: 不需要在表格中嵌入控件
3. **对现有功能侵入性小**: 主要是扩展，不是重构
4. **用户价值明确**: 快速过滤不想要的匹配
5. **为P1打基础**: P1的快速切换也需要过滤逻辑

```python
def filter_by_threshold(self, min_similarity: float) -> List[MatchCandidate]:
    """
    返回相似度≥min_similarity的候选

    Args:
        min_similarity: 最小相似度阈值 (0-1)

    Returns:
        过滤后的候选列表
    """
    all_candidates = self.get_all_candidates()
    return [c for c in all_candidates if c.similarity_score >= min_similarity]

def get_filtered_candidates(
    self,
    min_similarity: float = 0.0,
    preserve_current: bool = True
) -> List[MatchCandidate]:
    """
    获取过滤后的候选，可选择保留当前选中

    Args:
        min_similarity: 最小相似度阈值
        preserve_current: 是否保留当前选中（即使低于阈值）

    Returns:
        过滤后的候选列表
    """
    filtered = self.filter_by_threshold(min_similarity)

    if preserve_current and self.current_match:
        if self.current_match not in filtered:
            filtered.insert(0, self.current_match)

    return filtered

def auto_select_best_within_threshold(self, min_similarity: float) -> Optional[MatchCandidate]:
    """
    自动选择阈值内最佳候选

    如果当前选中低于阈值，自动选择最高相似度的候选

    Args:
        min_similarity: 最小相似度阈值

    Returns:
        选中的候选，如果没有则返回None
    """
    if self.current_match and self.current_match.similarity_score >= min_similarity:
        return self.current_match

    filtered = self.filter_by_threshold(min_similarity)
    if filtered:
        return max(filtered, key=lambda x: x.similarity_score)

    return None
```

```python
def set_match_mode(self, mode: MatchMode):
    """
    设置匹配模式并刷新表格

    Args:
        mode: 匹配模式枚举
    """
    from pyqt_ui.config import MATCH_THRESHOLDS

    self.current_match_mode = mode

    # 根据模式设置阈值
    self.current_threshold = MATCH_THRESHOLDS.get(mode, 0.60)

    # 更新按钮状态
    self.strict_btn.setChecked(mode == MatchMode.STRICT)
    self.standard_btn.setChecked(mode == MatchMode.STANDARD)
    self.loose_btn.setChecked(mode == MatchMode.LOOSE)

    # 如果有搜索结果，刷新表格（应用过滤）
    if hasattr(self, 'current_batch_search_result') and self.current_batch_search_result:
        self.populate_batch_results_table(
            self.current_batch_search_result,
            min_similarity=self.current_threshold
        )

        # 更新状态栏
        total_matched = self.current_batch_search_result.get_match_count()
        self.statusBar().showMessage(
            f"已应用{mode.value}模式(≥{self.current_threshold:.0%})，"
            f"{total_matched}首歌有匹配",
            5000
        )

    # 保存用户偏好
    self.save_match_preferences()

def toggle_advanced_options(self):
    """切换高级选项显示"""
    is_visible = not self.advanced_options_widget.isVisible()
    self.advanced_options_widget.setVisible(is_visible)
    self.toggle_advanced_btn.setText(
        "高级选项 ▲" if is_visible else "高级选项 ▼"
    )

def on_custom_threshold_changed(self, value: int):
    """
    自定义阈值滑块值变化

    Args:
        value: 阈值百分比 (0-100)
    """
    threshold = value / 100.0
    self.current_threshold = threshold
    self.threshold_label.setText(f"{value}%")

    # 自动切换到自定义模式
    from pyqt_ui.config import MatchMode
    self.current_match_mode = MatchMode.CUSTOM

    # 刷新表格
    if hasattr(self, 'current_batch_search_result') and self.current_batch_search_result:
        self.populate_batch_results_table(
            self.current_batch_search_result,
            min_similarity=threshold
        )

    self.save_match_preferences()
```

```python
def load_match_preferences(self) -> None:
    """
    从QSettings加载用户偏好

    在__init__()中调用
    """
    from PyQt6.QtCore import QSettings
    from pyqt_ui.config import MatchMode, DEFAULT_MATCH_MODE, DEFAULT_MATCH_THRESHOLD

    settings = QSettings("MusicDownloader", "BatchDownload")

    # 加载模式
    mode_str = settings.value("match_mode", DEFAULT_MATCH_MODE.value)
    try:
        self.current_match_mode = MatchMode(mode_str)
    except ValueError:
        self.current_match_mode = DEFAULT_MATCH_MODE

    # 加载自定义阈值
    custom_threshold = settings.value("custom_threshold", DEFAULT_MATCH_THRESHOLD)
    self.current_threshold = float(custom_threshold)

    # 加载权重
    weights = settings.value("similarity_weights", DEFAULT_SIMILARITY_WEIGHTS)
    self.similarity_weights = weights

def save_match_preferences(self) -> None:
    """
    保存用户偏好到QSettings

    在每次修改后调用
    """
    from PyQt6.QtCore import QSettings

    settings = QSettings("MusicDownloader", "BatchDownload")

    # 保存模式
    settings.setValue("match_mode", self.current_match_mode.value)

    # 保存阈值
    if self.current_match_mode == MatchMode.CUSTOM:
        settings.setValue("custom_threshold", self.current_threshold)

    # 保存权重
    settings.setValue("similarity_weights", self.similarity_weights)
```

```python
def quick_switch_to_candidate(
    self,
    original_line: str,
    new_candidate: MatchCandidate
):
    """
    快速切换到指定候选（通过菜单）

    Args:
        original_line: 原始输入行
        new_candidate: 要切换到的新候选
    """
    if not hasattr(self, "current_batch_search_result"):
        QMessageBox.warning(self, "错误", "没有找到批量搜索结果")
        return

    song_match = self.current_batch_search_result.matches.get(original_line)
    if not song_match:
        return

    # 记录旧候选（用于撤销）
    old_candidate = song_match.current_match

    # 执行切换
    from .batch.models import MatchSource
    song_match.switch_to_candidate(new_candidate, MatchSource.USER_SELECTED)

    # 刷新表格
    current_threshold = getattr(self, 'current_threshold', 0.0)
    self.populate_batch_results_table(
        self.current_batch_search_result,
        min_similarity=current_threshold
    )

    # 更新状态栏
    source_short = new_candidate.source.replace('MusicClient', '')
    self.statusBar().showMessage(
        f"✓ 已切换到: {new_candidate.song_name} - {new_candidate.singers} "
        f"({source_short}, {new_candidate.similarity_score:.2%})",
        4000
    )

    # 可选：添加到撤销历史
    self._add_to_undo_history(original_line, old_candidate, new_candidate)
```

```python
def __init__(self):

    # 撤销历史

def _add_to_undo_history(
    self,
    original_line: str,
    old_candidate: MatchCandidate,
    new_candidate: MatchCandidate
):
    """添加切换操作到历史记录"""
    self.switch_history.append((original_line, old_candidate, new_candidate))

    # 限制历史大小
    if len(self.switch_history) > self.max_history_size:
        self.switch_history.pop(0)

def undo_last_switch(self):
    """撤销最后一次切换"""
    if not self.switch_history:
        QMessageBox.information(self, "提示", "没有可撤销的操作")
        return

    original_line, old_candidate, new_candidate = self.switch_history.pop()

    song_match = self.current_batch_search_result.matches.get(original_line)
    if song_match:
        from .batch.models import MatchSource
        song_match.switch_to_candidate(old_candidate, MatchSource.USER_SELECTED)

        # 刷新表格
        current_threshold = getattr(self, 'current_threshold', 0.0)
        self.populate_batch_results_table(
            self.current_batch_search_result,
            min_similarity=current_threshold
        )

        self.statusBar().showMessage(
            f"↶ 已撤销切换: {old_candidate.song_name} - {old_candidate.singers}",
            3000
        )
```

## 风险评估与缓解

### 技术风险

#### 风险1: 表格性能问题

**描述**: 大量行（200+）时，每行添加控件可能导致卡顿

**影响**: 中等
**概率**: 低

**缓解措施**:
1. 使用懒加载：只创建可见行的控件
2. 虚拟化表格：考虑使用QTableView替代QTableWidget
3. 性能测试：200首歌每首20个候选，测量渲染时间
4. 添加加载指示器：即使操作很快，也给用户反馈

**应急方案**:
- 如果性能问题严重，禁用快速切换按钮，只保留对话框切换
- 使用分页显示（每页50首）

---

#### 风险2: 状态同步复杂度

**描述**: 快速切换、过滤、对话框切换之间可能出现状态不一致

**影响**: 高

#### 风险3: 用户偏好冲突

**描述**: 用户设置"严格90%"但当前选中的候选只有85%

**影响**: 低
**概率**: 高

**缓解措施**:

2. **自动选择更好候选**: 提示用户有更好的候选
3. **视觉警告**: 低相似度的候选用特殊颜色标记
4. **用户确认**: 首次切换到严格模式时显示提示

**实现示例**:
```python
def set_match_mode(self, mode: MatchMode):
    # ... 应用过滤 ...

    # 检查是否有歌曲当前选中低于阈值
    below_threshold_count = 0
    for song_match in self.current_batch_search_result.matches.values():
        if (song_match.current_match and
            song_match.current_match.similarity_score < self.current_threshold):
            below_threshold_count += 1

    # 如果有歌曲低于阈值，显示警告
    if below_threshold_count > 0:
        QMessageBox.warning(
            self,
            "匹配质量提示",
            f"当前有{below_threshold_count}首歌的选中匹配低于{self.current_threshold:.0%}阈值。\n"
```
